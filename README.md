# SDG 4 Quality Education - E-Learning Mobile App

## 📱 Project Overview

This project is a mobile application prototype designed to support **Sustainable Development Goal 4 (SDG 4): Quality Education**. The application aims to provide learners with easy access to educational resources, personalized learning paths, and user-friendly learning tools. This prototype was developed using **Figma** as part of a software engineering project following the Software Development Life Cycle (SDLC).

---

## 🎯 Purpose

The application addresses the United Nations' SDG 4 goal to "ensure inclusive and equitable quality education and promote lifelong learning opportunities for all" by 2030. The prototype demonstrates how mobile technology can enhance access to education, particularly in regions like Sierra Leone where initiatives like the Free Quality School Education (FQSE) are being implemented.

---

## ✨ Key Features

### Functional Requirements

| Feature | Description |
|---------|-------------|
| **User Registration** | Users can create accounts to access the platform |
| **User Login** | Secure login system for registered users |
| **Learning Materials** | Access to educational articles, videos, and resources |
| **User Profile** | Manage personal information and learning preferences |
| **Personalized Learning Paths** | Customized learning experiences based on progress |
| **Reporting System** | Report educational issues with photos and GPS data |
| **Notifications** | Automated alerts for deadlines and assignments |
| **Community Features** | Join educational events and interact with posts |
| **Dashboard** | Quick overview of learning activities and progress |

### Non-Functional Requirements

- **Performance**: Core screens load within 3 seconds
- **Security**: User credentials are encrypted
- **Usability**: Intuitive interface with high-contrast buttons
- **Compatibility**: Supports standard Android screen sizes
- **Reliability**: Maintains high system uptime
- **Accessibility**: Text meets accessibility standards

---

## 🏗️ System Architecture

The application follows a layered architecture:

1. **User Interface Layer** - Figma-designed screens and user interactions
2. **Authentication Layer** - User login, registration, and session management
3. **Core Business Components**:
   - Student Management
   - Course Management
   - Communication Module
   - Progress & Reports
   - Security & Backup
4. **Central Database** - Persistent data storage for all components

## 🛠️ Development Process

### SDLC Model Used: **Waterfall Model**

**Advantages:**
- Structured and disciplined approach
- Clear milestones and deliverables
- Comprehensive documentation

**Challenges Addressed:**
- Requirements were clearly defined upfront
- Design and testing phases are sequential
- Suitable for educational app with stable requirements

### Phases Followed:

1. **Requirements Gathering** - Defined functional and non-functional requirements
2. **Planning** - Set timelines and resources
3. **Design** - Developed UI/UX in Figma
